# global-tidy — 全局知识库整理 Skill

跨项目记忆分层归位、全局配置审查、权限白名单优化、历史对话提炼。定期使用，非高频。

## 为什么需要

多个项目并行开发时，信息容易散落：
- 在项目 A 里说的全局偏好，项目 B 的 agent 不知道
- 踩过的坑，换个项目又踩一遍
- 权限弹窗越来越多，打断工作流
- 全局配置积累过期内容

global-tidy 定期扫描历史对话，把信息按正确层级归档：**全局的归全局，项目的归项目，子项目的归子项目**。

## 与 neat-freak 的关系

| | neat-freak | global-tidy |
|------|------------|-------------|
| 范围 | 单项目内部 | 跨项目全局 |
| 关注 | 项目 docs/ + CLAUDE.md + memory | 多项目间信息分配 + 全局配置 + 权限 |
| 频率 | 开发节点触发 | 一段时间用一次 |
| 方向 | 深度 | 广度 |

两者互补，不重复。

## 功能

1. **增量扫描** — 读取上次整理以来的历史对话
2. **信息提取** — 自动识别全局偏好、项目决策、踩坑记录
3. **权限审查** — 分析权限弹窗，生成三级白名单建议
4. **分层归档** — 按全局/项目/子项目归位到正确位置
5. **自检验证** — 逐项检查分层正确性、完整性、质量

## 安装

```powershell
# 克隆仓库
git clone https://github.com/2021291696/global-tidy.git

# 复制到 Claude Code skills 目录
Copy-Item -Recurse global-tidy/ "$env:USERPROFILE\.claude\skills\global-tidy"
```

## 使用

```
# 命令
/global-tidy

# 或说
"全局整理"
"整理全局"
"全局同步"
```

首次运行会询问回溯范围（最近一周/一月/全部）。之后每次只扫描增量。

## 权限白名单三级

每次整理时分析权限弹窗模式，生成建议：

| 等级 | 范围 | 适合 |
|------|------|------|
| L1 保守 | 只读操作（git status、Read、Grep、Glob） | 新手 |
| L2 适中 | L1 + 可逆写入（git commit、Write、Edit、测试） | 有经验 |
| L3 宽松 | L2 + 网络/删除（git push、npm install、rm） | 高级用户 |

## 设计理念

- **信息分层归位**：不是全堆全局，也不是全塞项目，每条信息找到最合适的那一层
- **保守下沉**：不确定放哪层时，放项目级不轻易放全局（全局污染代价远大于项目遗漏）
- **增量不扰民**：只扫描上次整理后的新对话，不重复扫全量
- **分批 + 自检**：先整理再做自检，不边做边查
